🔘 Button Hierarchy & Actions

Hiérarchie des boutons et actions principales/secondaires

Primary Actions (CTA)

  • Dashboard KASH: "Voir mon plan", "Télécharger mon rapport", "Commencer quiz"
  • Couleur: Bleu principal (#1E3A8A)
  • Taille: Large, position visible, sticky si nécessaire
  • Usage: Actions principales, conversion, engagement

Secondary Actions

  • Actions complémentaires: "Partager", "Sauvegarder", "Filtrer"
  • Couleur: Gris ou violet (#8B5CF6)
  • Taille: Medium, moins proéminentes
  • Usage: Options, configuration, navigation secondaire

Destructive Actions

  • Actions irréversibles: "Supprimer", "Annuler", "Dismiss"
  • Couleur: Rouge (#EF4444) ou Orange (#F59E0B)
  • Confirmation: Modal de confirmation si irreversible
  • Usage: Actions critiques, suppression, annulation

📚 Guidelines d'Implémentation

🎯
Button Hierarchy: Utiliser le bleu principal pour les actions de conversion, violet pour les options secondaires, et rouge/orange uniquement pour les actions destructives avec confirmation modale.
💬
Feedback Messages: Toujours positifs et encourageants pour Lina, concis et actionnables pour M. Karim. Utiliser des icônes cohérentes et des couleurs sémantiques.
📝
Form Validation: Inline validation non-blocking avec messages constructifs. Sauvegarde automatique pour éviter la perte de données, micro-feedback pour chaque interaction.
🧭
Navigation Cohérence: Maintenir la même palette de couleurs et typographie entre les vues Lina et M. Karim. Mobile-first avec enhancement progressif pour desktop.
♿
Accessibilité Universelle: Tous les patterns doivent être keyboard navigables, screen reader compatibles, et respecter WCAG AA minimum avec AAA pour les éléments critiques.
